# ==== 🥇 1. CIC-Bell-DNS-EXF-2021 — основной (core training dataset) ====== #
Ссылка: https://www.unb.ca/cic/datasets/dns-exf-2021.html
Факт: Специально создан для обнаружения DNS data exfiltration и tunneling
Содержит:
benign + light + heavy атаки
stateful (временные) + stateless признаки
Для чего нужен: Основное обучение модели
Обучение: ML (Random Forest, XGBoost)DL (CNN), LSTM (важно → есть time-window признаки)
Почему критичен:
Единственный dataset, который:
прямо моделирует exfiltration сценарии
поддерживает sequence modeling (LSTM)

# ==== 🥈 2. BCCC-CIC-Bell-DNS-2024 — финальная проверка (evaluation dataset) ====== #
Ссылка: https://www.yorku.ca/research/bccc/
Факт: Объединяет DNS-2021 + EXF-2021
Добавляет:
6 типов атак
flow-based + application-level признаки, 120 признаков (расширенный feature space)
Для чего нужен: Финальное тестирование модели (generalization)
Проверка: как модель работает на новых данных, устойчивость к разным типам атак
Почему важен:
Это наиболее современный и сложный dataset
Используеться как: research-level benchmark, проверка “не переобучилась ли модель”

# ====  🥉 3. CIC-Bell-DNS-2021 — baseline + нормальный трафик ====== #
Ссылка: https://www.unb.ca/cic/datasets/dns-2021.html
Факт: ~1 млн доменов, ~99% benign (реалистичный интернет-трафик)
содержит: phishing, malware, spam
Для чего нужен:
Baseline (эталон нормального поведения)
Проверка: False Positives (ложные срабатывания)
Балансировка: добавление real-world benign трафика
Почему важен:
Без него модель: будет переобучена на атаки, даст много false positives

# ==== 🟡 4. Mendeley DNS Exfiltration Dataset — robustness / реалистичность ====== #
Ссылка: https://data.mendeley.com/datasets/c4n7fckkz3/3
Факт (обобщение): DNS exfiltration трафик, ближе к реальным условиям (ISP / реальные паттерны)
Для чего нужен:
Проверка: насколько модель устойчива к новым данным,
Cross-dataset validation:
train на CIC → test на Mendeley
Почему важен:
показывает: работает ли модель вне лаборатории

# ==== 🟡 5. Kaggle DNS Tunneling Dataset — быстрые эксперименты ====== #
Ссылка: https://www.kaggle.com/datasets/daumel/dns-tunneling-dataset
Факт (обобщение):
Синтетический dataset, несколько инструментов tunneling
Для чего нужен:
Быстрые задачи: прототипирование модели, feature engineering
Data augmentation: увеличение разнообразия атак
Почему ограничен: синтетика → хуже переносится на real-world

# ===== Правильное распределение ролей: ====== #
TRAIN:
- CIC-Bell-DNS-EXF-2021 → обучение
VALIDATION:
- CIC-Bell-DNS-2021 → контроль ошибок
TEST:
- BCCC-CIC-Bell-DNS-2024→ финальная проверка
- Mendeley DNS → проверка реалистичности
EXPERIMENTS:
- Kaggle DNS → быстрые тесты

# ===== ✅ Правильная схема обучения ==== #
TRAIN:
- EXF-2021   → attack class
- DNS-2021   → benign class
VALIDATION:
- часть DNS-2021 (или отдельный split)
TEST:
- BCCC-2024
- Mendeley
