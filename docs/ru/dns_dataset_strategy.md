# Стратегия DNS-датасетов для проекта обнаружения DNS Exfiltration и DNS Tunneling

## 1. Назначение документа

Документ фиксирует набор датасетов для обучения, валидации, тестирования и экспериментов в проекте по обнаружению DNS-эксфильтрации данных и DNS-туннелирования.

Основная цель — не смешивать роли датасетов и использовать каждый набор данных по назначению:

- **TRAIN** — обучение модели на атаках и нормальном трафике.
- **VALIDATION** — контроль качества модели и ложных срабатываний.
- **TEST** — финальная проверка обобщающей способности модели.
- **EXPERIMENTS** — быстрые эксперименты, прототипирование и проектирование признаков.

---

# 2. Обязательные датасеты

## 1. CIC-Bell-DNS-EXF-2021

### Роль

**Основной датасет для обучения модели атакующему поведению.**

### Ссылка

<https://www.unb.ca/cic/datasets/dns-exf-2021.html>

### Факт

CIC-Bell-DNS-EXF-2021 специально создан для обнаружения:

- DNS-эксфильтрации данных;
- DNS-туннелирования;
- легких сценариев атак;
- тяжелых сценариев атак.

### Содержит

- легитимный трафик;
- легкие атаки;
- тяжелые атаки;
- признаки с учетом состояния;
- признаки без учета состояния;
- временные признаки, пригодные для моделирования последовательностей.

### Для чего нужен

Используется как **основной обучающий датасет** для обучения модели обнаружению DNS-эксфильтрации и DNS-туннелирования.

### Подходящие модели

| Тип модели | Применение |
|---|---|
| Random Forest | базовая ML-модель |
| XGBoost | сильный baseline для табличных признаков |
| CNN | анализ локальных паттернов признаков |
| LSTM | анализ временных окон и последовательностей |

### Почему критичен

Это ключевой датасет, потому что он:

- напрямую моделирует сценарии DNS-эксфильтрации;
- содержит разные уровни атак;
- поддерживает моделирование последовательностей;
- подходит для LSTM за счет признаков временных окон.

### Использование в проекте

```text
TRAIN:
- класс атак
- DNS-эксфильтрация
- DNS-туннелирование
```

## 2. BCCC-CIC-Bell-DNS-2024

### Роль

**Финальная проверка модели и исследовательский benchmark.**

### Ссылка

<https://www.yorku.ca/research/bccc/ucs-technical/cybersecurity-datasets-cds/>

### Факт

BCCC-CIC-Bell-DNS-2024 объединяет данные из:

- DNS-2021;
- EXF-2021.

Также он добавляет расширенное пространство признаков и дополнительные типы атак.

### Содержит

- 6 типов атак;
- flow-based признаки;
- application-level признаки;
- 120 признаков;
- расширенное пространство признаков.

### Для чего нужен

Используется для финального тестирования модели:

- проверка обобщающей способности;
- проверка устойчивости к новым данным;
- проверка работы на разных типах атак;
- выявление переобучения.

### Почему важен

Это самый современный и сложный датасет из списка. Его целесообразно использовать не для основного обучения, а как независимый тестовый benchmark.

### Использование в проекте

```text
TEST:
- финальная оценка
- проверка обобщения
- проверка переобучения
```

---

## 3. CIC-Bell-DNS-2021

### Роль

**Baseline нормального DNS-трафика и контроль ложных срабатываний.**

### Ссылка

<https://www.unb.ca/cic/datasets/dns-2021.html>

### Факт

CIC-Bell-DNS-2021 содержит около 1 млн доменов, при этом примерно 99% данных относятся к легитимному трафику.

### Содержит

- легитимный трафик;
- phishing;
- malware;
- spam;
- реалистичное распределение интернет-трафика.

### Для чего нужен

Датасет нужен для формирования понимания нормального поведения DNS-трафика.

Основные задачи:

- baseline нормального поведения;
- обучение класса benign;
- контроль ложных срабатываний;
- балансировка обучающей выборки;
- добавление real-world benign traffic.

### Почему важен

Без CIC-Bell-DNS-2021 модель будет слишком сильно ориентирована на атаки. Это приведет к большому числу ложных срабатываний на реальном DNS-трафике.

### Использование в проекте

```text
TRAIN:
- класс benign

VALIDATION:
- контроль false positives
- проверка поведения нормального трафика
```

# 3. Дополнительные датасеты

## 4. Mendeley DNS Exfiltration Dataset

### Роль

**Проверка реалистичности и устойчивости.**

### Ссылка

<https://data.mendeley.com/datasets/c4n7fckkz3/3>

### Факт

Датасет содержит DNS exfiltration traffic и ближе к реальным условиям по сравнению с чисто лабораторными наборами данных.

### Для чего нужен

Используется для проверки устойчивости модели к новым данным.

Основной сценарий:

```text
TRAIN:
- CIC datasets

TEST:
- Mendeley DNS Exfiltration Dataset
```

### Проверяет

- переносимость модели;
- устойчивость к новым паттернам;
- способность работать вне лабораторных условий;
- качество cross-dataset validation.

### Почему важен

Mendeley DNS Exfiltration Dataset показывает, насколько модель применима к данным, которые отличаются от обучающего распределения.

---

## 5. Kaggle DNS Tunneling Dataset

### Роль

**Быстрые эксперименты и прототипирование.**

### Ссылка

<https://www.kaggle.com/datasets/daumel/dns-tunneling-dataset>

### Факт

Датасет является синтетическим и содержит DNS tunneling traffic, сгенерированный несколькими tunneling-инструментами.

### Для чего нужен

Подходит для:

- быстрых экспериментов;
- проектирования признаков;
- первичного прототипирования;
- data augmentation;
- проверки идей до запуска на основных датасетах.

### Ограничение

Датасет синтетический, поэтому его результаты хуже переносятся на реальный трафик. Его не стоит использовать как основной источник истины для оценки качества модели.

### Использование в проекте

```text
EXPERIMENTS:
- быстрые проверки
- проектирование признаков
- augmentation
```


```text
-> контроль false positives
-> настройка threshold
```

## TEST

```text
-> финальный benchmark
-> проверка generalization
-> устойчивость к нескольким типам атак
```

```text
Mendeley DNS Exfiltration Dataset
-> cross-dataset validation
-> проверка real-world паттернов
```

```text
Kaggle DNS Tunneling Dataset
-> прототип
-> проектирование признаков
-> быстрое сравнение моделей
```

# 6. Итоговая логика использования

## Основная идея

Модель должна учиться не только распознавать атаки, но и понимать нормальное поведение DNS-трафика.

Поэтому обучение строится так:

```text
EXF-2021 учит модель видеть атаки.
DNS-2021 учит модель понимать нормальный DNS-трафик.
BCCC-2024 проверяет, не переобучилась ли модель.
Mendeley проверяет применимость к более реалистичным данным.
Kaggle используется только для быстрых экспериментов.
```
